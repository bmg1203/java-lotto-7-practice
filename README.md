# java-lotto-precourse

# 로또

## 프로젝트 설명
이 프로젝트는 우아한테크코스의 프리코스 3주차 미션으로, 로또를 구현합니다. 사용자는 원하는 금액만큼의 로또를 구매할 수 있고, 구매한 매수의 로또의 당첨결과와 수익률을 알려줍니다.

## 기능 요구사항
간단한 로또 발매기를 구현한다.

- 로또 번호의 숫자 범위는 1~45까지이다.
- 1개의 로또를 발행할 때 **중복되지 않는 6개의 숫자**를 뽑는다.
- 당첨 번호 추첨 시 **중복되지 않는 숫자 6개**와 **보너스 번호 1개**를 뽑는다.
- 당첨은 1등부터 5등까지 있다. 당첨 기준과 금액은 아래와 같다.  
  **1등: 6개 번호 일치 / 2,000,000,000원**  
  **2등: 5개 번호 + 보너스 번호 일치 / 30,000,000원**  
  **3등: 5개 번호 일치 / 1,500,000원**  
  **4등: 4개 번호 일치 / 50,000원**  
  **5등: 3개 번호 일치 / 5,000원**
- 로또 구입 금액을 입력하면 구입 금액에 해당하는 만큼 로또를 발행해야 한다.
- 로또 1장의 가격은 **1,000원**이다.
- 당첨 번호와 보너스 번호를 입력받는다.
- 사용자가 구매한 로또 번호와 당첨 번호를 비교하여 당첨 내역 및 수익률을 출력하고 로또 게임을 종료한다.
- 사용자가 잘못된 값을 입력할 경우, `IllegalArgumentException`을 발생시키고, `[ERROR]`로 시작하는 에러 메시지를 출력 후 그 부분부터 입력을 다시 받는다.
- `Exception`이 아닌 `IllegalArgumentException`, `IllegalStateException` 등과 같은 명확한 유형을 처리한다.

## 동작 방식
1. 로또 금액 입력
2. 당첨 번호 입력
3. 보너스 번호 입력
4. 금액만큼 로또 수량, 번호 발급(오름차순)
5. 당첨 내역 출력
6. 수익률 계산

&#8251; **에러 문구는 [ERROR]로 시작**

## 예외상황

1. 로또 금액이 1000원 단위로 입력되지 않았을 경우
2. 당첨 번호 추첨 시 범위가 1-45를 벗어날 경우
3. 당첨 번호 추첨 시 & 보너스 번호 추첨 시 중복이 존재할 경우


## 기능별 클래스 설명
### Application 
- 프로그램의 **진입점 역할**을 하며, `LottoMachine` 클래스를 호출하여 전체 로또 발매 및 당첨 확인 과정을 시작한다.
### LottoMachine 
- **프로그램의 전체 흐름을 관리**합니다.
  1. 로또 구입 금액 입력
  2. 로또 번호 생성 및 출력
  3. 당첨 번호 입력
  4. 당첨 결과 확인 및 출력
### LottoGenerator 
- **로또 티켓을 생성**한다.
- 구입 금액을 기준으로 필요한 수량의 로또 티켓을 생성하고, 1~45 사이의 중복되지 않는 6개 번호로 구성한다.
- `Randoms.pickUniqueNumbersInRange()` 메서드를 이용하여 랜덤한 번호를 생성한다.
### Lotto 
- **로또 티켓의 번호를 관리**하는 클래스다.
- 6개의 번호로 구성되며, 오름차순으로 정렬한다.
- **번호 유효성 검사** : 번호는 6개여야 하며, 유효하지 않으면 `IllegalArgumentException`을 발생시킨다.
### WinningNumber 
- **당첨 번호와 보너스 번호를 관리**하는 클래스다.
- 입력된 6개의 당첨 번호와 보너스 번호가 중복되지 않도록 검사한다.
- **유효성 검사** : 당첨 번호는 1~45 사이의 값이며, 보너스 번호와 중복이 없어야 한다.
### LottoResultChecker 
- **로또 당첨 여부를 확인**하는 클래스다.
- 각 로또 티켓의 번호와 당첨 번호를 비교하여 등수를 결정한다.
- `Rank Enum`을 이용해 당첨 등수를 판별하고, 각 로또 티켓의 결과를 `LottoResult`에 저장한다.
### Rank (Enum) 
- **당첨 등수와 상금을 관리**하는 `Enum`이다.
- 각 등수는 일치하는 번호 개수와 보너스 번호 여부에 따라 정의된다.
- `findRank()` 메서드를 통해 일치하는 번호 개수와 보너스 여부에 따라 등수를 반환한다.
### LottoResult 
- **로또 게임의 전체 당첨 결과를 관리**하는 클래스다.
- 각 등수에 따른 당첨 개수를 기록하고, 사용자가 구입한 총 금액 대비 수익률을 계산한다.
- `calculateProfitRate()` 메서드를 통해 수익률 계산을 수행한다.
### LottoView 
- **사용자와의 입출력 인터페이스를 담당**하는 클래스다.
- 로또 구입 금액, 당첨 번호, 보너스 번호 등의 입력을 처리한다. 
- 구매한 로또 티켓과 당첨 결과 및 수익률을 출력한다.



## 입출력 요구사항

### 입력
1. 로또 구입 금액을 입력 받는다. 구입 금액은 1,000원 단위로 입력 받으며 1,000원으로 나누어 떨어지지 않는 경우 예외 처리한다.
    - 예시 : `14000`
2. 당첨 번호를 입력 받는다. 번호는 쉼표(,)를 기준으로 구분한다.
    - 예시 : `1,2,3,4,5,6`
3. 보너스 번호를 입력 받는다.
    - 예시 : `7`

### 출력
1. 발행한 로또 수량 및 번호를 출력한다. 로또 번호는 오름차순으로 정렬하여 보여준다.
    ```
    8개를 구매했습니다.
   [8, 21, 23, 41, 42, 43]
   [3, 5, 11, 16, 32, 38]
   [7, 11, 16, 35, 36, 44]
   [1, 8, 11, 31, 41, 42]
   [13, 14, 16, 38, 42, 45]
   [7, 11, 30, 40, 42, 43]
   [2, 13, 22, 32, 38, 45]
   [1, 3, 5, 14, 22, 45]
    ```
   
2. 당첨 내역을 출력한다.
    ```
   3개 일치 (5,000원) - 1개
   4개 일치 (50,000원) - 0개
   5개 일치 (1,500,000원) - 0개
   5개 일치, 보너스 볼 일치 (30,000,000원) - 0개
   6개 일치 (2,000,000,000원) - 0개
   ```
   
3. 수익률은 소수점 둘째 자리에서 반올림한다. (ex. 100.0%, 51.5%, 1,000,000.0%)
    - 예시 : `총 수익률은 62.5%입니다.`
4. 예외 상황 시 에러 문구를 출력해야 한다. 단, 에러 문구는 "[ERROR]"로 시작해야 한다.
    - 예시 : `[ERROR] 로또 번호는 1부터 45 사이의 숫자여야 합니다.`
