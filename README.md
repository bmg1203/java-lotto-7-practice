## ✅ 기능 요구 사항 <a id="functional-requirements"></a>

간단한 로또 발매기를 구현한다.

- 로또 번호의 숫자 범위는 1~45까지이다.
- 1개의 로또를 발행할 때 중복되지 않는 6개의 숫자를 뽑는다.
- 당첨 번호 추첨 시 중복되지 않는 숫자 6개와 보너스 번호 1개를 뽑는다.
- 당첨은 1등부터 5등까지 있다. 당첨 기준과 금액은 아래와 같다.
    - 1등: 6개 번호 일치 / 2,000,000,000원
    - 2등: 5개 번호 + 보너스 번호 일치 / 30,000,000원
    - 3등: 5개 번호 일치 / 1,500,000원
    - 4등: 4개 번호 일치 / 50,000원
    - 5등: 3개 번호 일치 / 5,000원
- 로또 구입 금액을 입력하면 구입 금액에 해당하는 로또를 발행해야 한다.
- 로또 1장의 가격은 1,000원이다.
- 당첨 번호와 보너스 번호를 입력 받는다.
- 사용자가 구매한 로또 번호와 당첨 번호를 비교하여 당첨 내역 및 수익률을 출력하고 로또 게임을 종료한다.
- 사용자가 잘못된 값을 입력할 경우 IllegalArgumentException을 발생시키고, "[ERROR]"로 시작하는 에러 메시지를 출력 후 그 부분부터 입력을 다시 받는다.
    - Exception이 아닌 IllegalArgumentException, IllegalStateException 등과 같은 명확한 유형을 처리한다. 

## ✅ 입출력 요구 사항 <a id="io-requirements"></a>

#### - 로또 구입 금액을 입력 받는다. 구입 금액은 1,000원 단위로 입력 받으며 1,000원으로 나누어 떨어지지 않는 경우 예외 처리한다.

## ✅ 프로그래밍 요구 사항 1 <a id="programming-requirements1"></a>

- JDK 21 버전에서 실행 가능해야 한다.
- 프로그램 실행의 시작점은 Application의 main()이다.
- build.gradle 파일은 변경할 수 없으며, 제공된 라이브러리 이외의 외부 라이브러리는 사용하지 않는다.
- 프로그램 종료 시 System.exit()를 호출하지 않는다.
- 프로그래밍 요구 사항에서 달리 명시하지 않는 한 파일, 패키지 등의 이름을 바꾸거나 이동하지 않는다.
- 자바 코드 컨벤션을 지키면서 프로그래밍한다.
    - 기본적으로 [Java Style Guide](https://github.com/woowacourse/woowacourse-docs/tree/main/styleguide/java)를 원칙으로 한다.

## ✅ 프로그래밍 요구 사항 2 <a id="programming-requirements2"></a>

- indent(인덴트, 들여쓰기) depth를 3이 넘지 않도록 구현한다. 2까지만 허용한다.
    - 예를 들어 while문 안에 if문이 있으면 들여쓰기는 2이다.
    - 힌트: indent(인덴트, 들여쓰기) depth를 줄이는 좋은 방법은 함수(또는 메서드)를 분리하면 된다.
- 3항 연산자를 쓰지 않는다.
- 함수(또는 메서드)가 한 가지 일만 하도록 최대한 작게 만들어라.
- JUnit 5와 AssertJ를 이용하여 정리한 기능 목록이 정상적으로 작동하는지 테스트 코드로 확인한다.
    - 테스트 도구 사용법이 익숙하지 않다면 아래 문서를 참고하여 학습한 후 테스트를 구현한다.
        - [JUnit5 User Guide](https://junit.org/junit5/docs/current/user-guide/)
        - [AssertJ User Guide](https://assertj.github.io/doc/)
        - [AssertJ Exception Assertions](https://www.baeldung.com/assertj-exception-assertion)
        - [Guide to JUnit5 Parameterized Tests](https://www.baeldung.com/parameterized-tests-junit-5)

## ✅ 프로그래밍 요구 사항 3 <a id="programming-requirements3"></a>

- 함수(또는 메서드)의 길이가 15라인을 넘어가지 않도록 구현한다.
    - 함수(또는 메서드)가 한 가지 일만 잘 하도록 구현한다.
- else 예약어를 쓰지 않는다.
    - else를 쓰지 말라고 하니 switch/case로 구현하는 경우가 있는데 switch/case도 허용하지 않는다.
    - 힌트: if 조건절에서 값을 return 하는 방식으로 구현하면 else를 사용하지 않아도 된다.
- Java Enum을 적용하여 프로그램을 구현한다.
- 구현한 기능에 대한 단위 테스트를 작성한다. 단, UI(System.out, System.in, Scanner) 로직은 제외한다.
    - 단위 테스트 작성이 익숙하지 않다면 LottoTest를 참고하여 학습한 후 테스트를 작성한다. 

### 라이브러리

- camp.nextstep.edu.missionutils에서 제공하는 Randoms 및 Console API를 사용하여 구현해야 한다.

    - 사용자가 입력하는 값은 camp.nextstep.edu.missionutils.Console의 readLine()을 활용한다.

    - 1에서 45 사이의 중복되지 않은 정수 6개 반환

## 📚 기능 구현 목록 <a id="implement-list"></a>

## 기능

- [x] 사용자 입력 값에 따른 로또 수량 표시
- [x] 로또 번호 출력
    - 1~45까지의 범위에서 중복되지 않는 랜덤 번호 6개를 출력한다.
    - 번호들은 오름차순으로 출력하며, 대괄호 안에 공백과 쉼표로 구분하여 출력한다.
- [x] 사용자가 입력한 당첨 및 보너스 번호에 따라 당첨 통계 출력
    - 1~5등까지 각각 몇 개가 당첨되었는지를 표시한다.
    - 당첨 판정 과정에서 당첨 번호와 보너스 번호는 따로 진행한다. 즉, 일치하는 수를 찾을 때 보너스 번호를 고려하지 않는다. (ex. 5개 일치, 보너스 볼 일치)
    - 구입 금액 대비 전체 수익률을 소수점 첫째 자리까지의 수로 나타낸다.

## 입력

- [x] 사용자
    -  로또 구입 금액을 1,000원 단위로 입력한다.
    -  당첨 번호 6개를 쉼표(,)로 구분하여 입력한다.
        - 1~45 범위의 중복되지 않는 6개의 숫자여야 한다.
        - 숫자와 쉼표 사이의 공백은 허용한다. (ex. "1, 2, 3, 4, 5, 6")
        - 하지만 수의 개수가 충족되어도 쉼표 사이의 공백 및 빈 문자열은 허용하지 않는다. (ex. "1,2,3,4,,5,6")
    - 보너스 번호를 1개 입력한다.
        - 1~45 범위의 하나의 숫자여야 한다.
        - 당첨 번호에 포함되지 않는 수여야 한다.

- [x] 시스템
    - 사용자가 입력한 로또 구입 금액을 보고 로또 수량을 계산한다.
        - 1,000원 단위가 아니라면 다시 입력 받도록 한다.
        - 로또 구입 금액은 최대 100,000원으로 제한한다.
    - 도출된 로또 수량 값만큼 로또 번호를 생성한다.
    - 사용자가 입력하는 당첨 번호가 1~45 범위 내의 중복되지 않는 수들인지 확인한다.
        - 아니라면 다시 입력 받도록 한다.
    - 사용자가 입력하는 보너스 번호가 1~45 범위 내의 하나의 수인지 확인한다.
        - 아니라면 다시 입력 받도록 한다.
    - 사용자가 입력한 당첨 및 보너스 번호로 당첨 통계를 내린다.

## 출력

- 사용자가 입력한 로또 구입 금액으로 처리된 로또 수량을 출력한다.
- 1~45 범위의 무작위로 선정된 로또 번호 6개를 오름차순으로 출력한다.
- 출력 양식은 대괄호 안에 쉼표와 공백으로 구분하여 출력한다. (ex. "[1, 2, 3, 4, 5, 6]")
- 당첨 통계는 1~5등에서 각각 몇 개가 당첨되었는지를 출력한 뒤, 총 수익률을 소수점 첫째 자리까지 출력한다. 

## 예외

- 입력 문자열이 공백 및 빈 문자열인 경우: "[ERROR] 입력 문자열이 공백이 되어서는 안 됩니다."
- 입력 문자열에 숫자와 쉼표를 제외하고 허용되지 않은 문자열이 포함되어 있는 경우: "[ERROR] 입력에 허용되지 않은 문자열이 포함되어 있습니다."
- 로또 구입 금액을 1,000원 단위로 입력하지 않은 경우: "[ERROR] 로또 구입 금액을 1,000원 단위로 입력해 주세요."
- 로또 구입 금액이 100,000원을 초과한 경우: "[ERROR] 1인당 구입 가능한 로또 금액은 최대 100,000원입니다."
- 사용자가 입력한 로또 번호가 6개가 아닌 경우: "[ERROR] 로또 번호는 6개여야 합니다."
- 사용자가 입력한 로또 번호 중 1~45 범위가 아닌 수가 포함된 경우: "[ERROR] 로또 번호는 1~45 범위의 숫자여야 합니다."
- 사용자가 입력한 로또 번호가 중복된 경우: "[ERROR] 로또 번호는 중복되지 않아야 합니다."
- 로또 번호 입력에 쉼표 사이의 공백 및 빈 문자열이 포함된 경우: "[ERROR] 로또 번호 입력에 쉼표 사이의 공백, 빈 문자열은 허용하지 않습니다."
- 사용자가 입력한 보너스 번호가 1~45 범위의 수가 아닌 경우: "[ERROR] 보너스 번호는 1~45 범위의 숫자여야 합니다."
- 보너스 번호가 로또 번호에 입력된 수인 경우: "[ERROR] 보너스 번호는 당첨 번호와 다른 수여야 합니다."

## 테스트
