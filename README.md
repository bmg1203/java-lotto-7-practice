# java-lotto-precourse

# 로또 발매기 기능 목록

1. **구입 금액 입력 및 검증**
  - 사용자는 로또 구입 금액을 입력할 수 있습니다.
  - 입력된 금액은 1,000원 단위여야 하며, 그렇지 않을 경우 예외가 발생합니다.
2. **로또 티켓 발행**
  - 구입 금액에 따라 로또 티켓을 자동으로 발행합니다.
  - 각 로또 티켓은 1,000원이며, 구입 금액에 따라 티켓 수가 결정됩니다.
3. **당첨 번호 입력 및 검증**
  - 사용자는 당첨 번호 6개를 입력합니다.
  - 입력된 번호는 1부터 45 사이의 숫자이며, 중복되지 않아야 합니다.
4. **보너스 번호 입력 및 검증**
  - 사용자는 보너스 번호 1개를 입력합니다.
  - 보너스 번호는 1부터 45 사이의 숫자이며, 당첨 번호와 중복되지 않아야 합니다.
5. **당첨 결과 계산**
  - 구매한 로또 티켓과 당첨 번호를 비교하여 등수를 계산합니다.
  - 등수는 일치하는 숫자의 개수와 보너스 번호 일치 여부에 따라 결정됩니다.
6. **수익률 계산 및 출력**
  - 총 수익률을 계산하여 출력합니다.
  - 수익률은 총 당첨 금액을 총 투자 금액으로 나눈 값입니다.
7. **예외 처리 및 에러 메시지 출력**
  - 입력값의 형식이나 범위가 유효하지 않을 경우 예외를 발생시키고, 적절한 에러 메시지를 출력합니다.

## 클래스별 설명

### 1. `lotto.application.LottoService`

- 로또 티켓 생성, 등수 계산, 수익률 계산 등의 핵심 로직을 담당하는 서비스 클래스입니다.

### 2. `lotto.domain.Lotto`

- 하나의 로또 티켓을 나타내는 도메인 클래스입니다.
- 6개의 로또 번호를 가지고 있으며, 번호 검증 로직을 포함합니다.

### 3. `lotto.domain.LottoTicket`

- 여러 개의 로또 티켓을 관리하는 클래스입니다.
- 구매한 모든 로또 티켓을 저장합니다.

### 4. `lotto.domain.Rank`

- 당첨 등수를 나타내는 Enum 클래스입니다.
- 일치하는 숫자 개수, 보너스 번호 일치 여부, 상금 금액을 포함합니다.

### 5. `lotto.domain.WinningLotto`

- 당첨 번호와 보너스 번호를 관리하는 클래스입니다.
- 당첨 번호와 보너스 번호를 기반으로 로또 티켓의 등수를 계산합니다.

### 6. `lotto.exception.LottoException`

- 로또 애플리케이션에서 발생하는 예외를 처리하는 유틸리티 클래스입니다.
- 다양한 예외 상황에 대한 에러 메시지를 포함하고 있습니다.

### 7. `lotto.presentation.LottoController`

- 프로그램의 전체 흐름을 제어하는 컨트롤러 클래스입니다.
- 입력 및 출력 컨트롤러를 사용하여 사용자와 상호 작용합니다.

### 8. `lotto.presentation.LottoInputController`

- 사용자로부터 입력을 받고, 입력값을 검증하는 역할을 담당합니다.

### 9. `lotto.presentation.LottoTicketController`

- 로또 티켓을 생성하고, 생성된 티켓을 출력하는 역할을 합니다.

### 10. `lotto.presentation.LottoResultController`

- 당첨 결과를 계산하고, 결과를 출력하는 역할을 담당합니다.

### 11. `lotto.util.LottoNumberGenerator`

- 로또 번호를 무작위로 생성하는 유틸리티 클래스입니다.

### 12. `lotto.util.Parser`

- 입력된 문자열을 숫자로 변환하는 파서 클래스입니다.
- 입력값의 형식을 검증하고 파싱합니다.

### 13. `lotto.util.ProfitCalculator`

- 수익률을 계산하는 유틸리티 클래스입니다.

### 14. `lotto.validator.BonusNumberValidator`

- 보너스 번호의 유효성을 검증하는 클래스입니다.

### 15. `lotto.validator.LottoValidator`

- 로또 번호와 구입 금액 등의 유효성을 검증하는 클래스입니다.

### 16. `lotto.view.InputView`

- 사용자로부터 입력을 받는 뷰 클래스입니다.

### 17. `lotto.view.OutputErrorView`

- 에러 메시지를 출력하는 뷰 클래스입니다.

### 18. `lotto.view.OutputLottoView`

- 구매한 로또 티켓을 출력하는 뷰 클래스입니다.

### 19. `lotto.view.OutputProfitRateView`

- 수익률을 출력하는 뷰 클래스입니다.

### 20. `lotto.view.OutputRankView`

- 당첨 통계를 출력하는 뷰 클래스입니다.

### 21. `lotto.Application`

- 프로그램의 진입점이며, `main` 메서드를 포함하고 있습니다.
- 예외 처리를 통해 프로그램이 중단되지 않고 에러 메시지를 출력하도록 합니다.

## 테스트 코드

### 1. `lotto.ApplicationTest`

- 애플리케이션의 전체 흐름에 대한 통합 테스트를 수행합니다.
- 예외 상황 및 정상 동작에 대한 테스트를 포함합니다.

### 2. `lotto.LottoServiceTest`

- `LottoService` 클래스의 기능을 테스트합니다.
- 구입 금액에 따른 로또 티켓 발행 및 예외 발생 여부를 검증합니다.

### 테스트 목록

- **구입 금액이 1,000원 단위가 아니면 예외가 발생한다.**
  - `generateLottoTickets` 메서드 호출 시 1,000원 단위가 아닌 금액이 입력되면 `IllegalArgumentException`이 발생하는지 테스트합니다.
- **구입 금액에 맞는 로또 티켓 수량이 발행된다.**
  - 입력된 구입 금액에 따라 올바른 수의 로또 티켓이 생성되는지 검증합니다.

### 3. `lotto.LottoTest`

- `Lotto` 클래스의 생성자와 번호 검증 로직을 테스트합니다.

### 테스트 목록

- **로또 번호의 개수가 6개가 아니면 예외가 발생한다.**
  - 번호의 개수가 6개가 아닐 경우 `IllegalArgumentException`이 발생하는지 테스트합니다.
- **로또 번호에 중복된 숫자가 있으면 예외가 발생한다.**
  - 로또 번호에 중복된 숫자가 포함되어 있으면 `IllegalArgumentException`이 발생하는지 테스트합니다.

### 4. `lotto.LottoTicketTest`

- `LottoTicket` 클래스의 기능을 테스트합니다.

### 테스트 목록

- **로또 번호 리스트가 6개가 아니면 예외가 발생한다.**
  - `Lotto` 객체 생성 시 번호의 개수가 6개가 아니면 예외가 발생하는지 테스트합니다.
- **로또 번호에 중복된 숫자가 있으면 예외가 발생한다.**
  - `Lotto` 객체 생성 시 중복된 번호가 있으면 예외가 발생하는지 테스트합니다.

### 5. `lotto.ProfitCalculatorTest`

- `ProfitCalculator` 클래스의 수익률 계산 로직을 테스트합니다.

### 테스트 목록

- **총 투자 금액과 당첨 금액을 통해 수익률을 계산한다.**
  - 투자 금액과 당첨 금액을 입력했을 때 올바른 수익률이 계산되는지 검증합니다.

### 6. `lotto.WinningLottoTest`

- `WinningLotto` 클래스의 생성자와 보너스 번호 검증 로직을 테스트합니다.

### 테스트 목록

- **보너스 번호가 당첨 번호와 중복되면 예외가 발생한다.**
  - 보너스 번호가 당첨 번호와 중복될 경우 `IllegalArgumentException`이 발생하는지 테스트합니다.
- **정상적인 당첨 번호와 보너스 번호를 설정한다.**
  - 유효한 당첨 번호와 보너스 번호로 객체를 생성할 수 있는지 테스트합니다.
