# java-lotto-precourse

## 목표
- Random 으로 로또 발매, 당첨 여부 및 수익률 계산을 구현한다.
- 기능을 세분하여 모듈화 한다.
- JUnit 5와 AssertJ를 이용하여 정리한 기능 목록이 정상적으로 작동하는지 테스트 코드로 확인한다.

## 기능 설명
1. 로또 구입 금액을 입력받는다.
   - 구입 금액은 1,000원 단위로 입력 받으며 1,000원으로 나누어 떨어지지 않는 경우 예외 처리한다.


2. 구입 금액에 해당하는 만큼 로또를 발행한다.
   - 로또 1장의 가격은 1,000원이다.
   - 1개의 로또를 발행할 때 중복되지 않는 6개의 숫자를 뽑는다.
   - 로또 번호의 숫자 범위는 1~45까지이다.


3. 발행한 로또 수량 및 번호를 출력한다.
   - 로또 번호는 오름차순으로 정렬하여 출력한다.


4. 당첨 번호와 보너스 번호를 입력받는다.
   - 당첨 번호는 중복되지 않는 숫자 6개와 보너스 번호 1개를 뽑는다.
   - 첫 줄에 당첨 번호를 쉼표(,)를 기준으로 구분하여 입력받는다.
   - 숫자와 쉼표 사이의 white space는 무시한다.
   - 그 다음 줄에 보너스 번호를 입력받는다.


5. 발행한 모든 로또의 당첨 내역과 수익률을 출력한다.
   - 당첨은 1등부터 5등까지 있다. 당첨 기준과 금액은 아래와 같다.
     - 1등: 6개 번호 일치 / 2,000,000,000원
     - 2등: 5개 번호 + 보너스 번호 일치 / 30,000,000원
     - 3등: 5개 번호 일치 / 1,500,000원
     - 4등: 4개 번호 일치 / 50,000원
     - 5등: 3개 번호 일치 / 5,000원
   - 수익률은 소수점 둘째 자리에서 반올림한다. (ex. 100.0%, 51.5%, 1,000,000.0%)


- 사용자가 잘못된 값을 입력할 경우 IllegalArgumentException을 발생시키고, "[ERROR]"로 시작하는 에러 메시지를 출력 후 그 부분부터 입력을 다시 받는다.
   - Exception이 아닌 IllegalArgumentException, IllegalStateException 등과 같은 명확한 유형을 처리한다.



##  Input Format
입력은 다음 3개의 줄이 순서대로 들어온다.
### 입력 1. 로또 구입 금액
| input              | 결과                                  | 비고            | 
|--------------------|-------------------------------------|---------------|
| 1000               | 출력 1 -> 입력 2 대기                     |               |
| 1100               | IllegalArgumentException -> 입력 1 대기 | 1000단위가 아닌 입력 |
|                    | IllegalArgumentException -> 입력 1 대기 | 빈 입력          |
| null               | IllegalArgumentException -> 입력 1 대기 | null 입력       |
| a                  | IllegalArgumentException -> 입력 1 대기 | 숫자가 아닌 입력     |
| 1.1                | IllegalArgumentException -> 입력 1 대기 | 정수가 아닌 입력     |
| 2<sup>32</sup>-1 < | IllegalArgumentException -> 입력 1 대기 | int overflow  |

### 입력 2. 당첨 번호
| input       | 결과                                  | 비고             | 
|-------------|-------------------------------------|----------------|
| 1,2,3,4,5,6 | 입력 3 대기                             |                |
| 1,2,  3     | 입력 3 대기                             | white space 허용 |
| 1,2,2       | IllegalArgumentException -> 입력 2 대기 | 중복된 숫자 입력      |
| 1,2,46       | IllegalArgumentException -> 입력 2 대기 | 범위를 넘는 숫자 입력   |
| a,b,c,d     | IllegalArgumentException -> 입력 2 대기 | 숫자가 아닌 입력      |
|             | IllegalArgumentException -> 입력 2 대기 | 빈 입력           |
| null        | IllegalArgumentException -> 입력 2 대기 | null 입력        |
| 2<sup>32</sup>-1 < | IllegalArgumentException -> 입력 1 대기 | int overflow  |

### 입력 3. 보너스 번호
| input | 결과                                  | 비고            | 
|-------|-------------------------------------|---------------|
| 1     | 출력 2                                |               |
| 1     | IllegalArgumentException -> 입력 3 대기 | 입력 2와 중복되는 숫자 |
| 46    | IllegalArgumentException -> 입력 3 대기 | 범위를 넘는 숫자 입력  |
|       | IllegalArgumentException -> 입력 3 대기 | 빈 입력          |
| null  | IllegalArgumentException -> 입력 3 대기 | null 입력       |
| a     | IllegalArgumentException -> 입력 3 대기 | 숫자가 아닌 입력      |
| 2<sup>32</sup>-1 < | IllegalArgumentException -> 입력 1 대기 | int overflow  |

## 구현 내역
### 1. Input parsing
- [x] **로또 구매 금액**
    - [x] 입력 메세지 출력
    - [x] 숫자 양끝의 white space 제거
    - [x] 1000의 양의 배수 확인
    - [x] 빈 이름 예외처리
    - [x] null 예외처리
    - [x] 숫자가 아닌 입력 예외처리
    - [x] overflow 예외처리
- [ ] **당첨 번호**
    - [ ] 입력 메세지 출력
    - [ ] 콤마(,) 기준 숫자 구분
    - [ ] 숫자 양끝의 white space 제거
    - [ ] 1 ~ 45 외 범위 예외처리
    - [ ] 중복 숫자 예외처리
    - [ ] overflow 예외처리
- [ ] **보너스 번호**
    - [ ] 입력 메세지 출력
    - [ ] 숫자 양끝의 white space 제거
    - [ ] 1 ~ 45 외 범위 예외처리
    - [ ] 중복 숫자 예외처리

### 2. Pick Lotto Numbers
- [ ] 구입 한 로또 수량 계산
- [ ] 로또 번호 생성
- [ ] 번호 오름차순 정렬

### 3. Statistic
- [ ] **당첨 조회**
    - [ ] 당첨 번호와 비교
    - [ ] 5개의 경우 2등 보너스 번호 비교

- [ ] **통계 출력**
    - [ ] 각 등수의 당첨 갯수 출력
