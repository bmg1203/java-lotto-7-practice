# 🎰 로또 발매기 기능 목록

1. **로또 구입 금액 입력 및 유효성 검사**
    - 사용자로부터 로또 구입 금액을 입력받는다.
    - 구입 금액이 **1,000원 단위**인지 확인한다.
    - 1,000원 단위가 아닐 경우 `[ERROR]`로 시작하는 에러 메시지를 출력하고 재입력 받는다.
      - 소숫점, 공백, 문자 등 숫자가 아닌 값은 모두 예외처리한다.

2. **로또 발행**
    - 구입 금액에 따라 **발행할 로또 개수를 계산**한다. (로또 한 장당 1,000원)
    - 각 로또마다 **중복되지 않는 6개의 번호**(1~45)를 생성한다.
    - 발행된 로또 번호는 **오름차순으로 정렬**하여 저장한다.

3. **발행된 로또 정보 출력**
    - 발행된 로또 개수와 각 로또 번호를 출력한다.

4. **당첨 번호 입력 및 유효성 검사**
    - 사용자로부터 당첨 번호를 입력받는다.
    - 입력된 당첨 번호는 **쉼표(,)**로 구분된 **중복되지 않는 6개의 숫자**여야 한다.
    - 각 번호가 **1~45 범위**인지 확인한다.
    - 유효하지 않은 번호가 입력될 경우 `[ERROR]`로 시작하는 에러 메시지를 출력하고 재입력 받는다.

5. **보너스 번호 입력 및 유효성 검사**
    - 사용자로부터 보너스 번호를 입력받는다.
    - 입력된 보너스 번호는 **1~45 범위의 숫자**여야 하며, **당첨 번호와 중복되지 않아야** 한다.
    - 유효하지 않은 번호가 입력될 경우 `[ERROR]`로 시작하는 에러 메시지를 출력하고 재입력 받는다.

6. **당첨 결과 확인**
    - 각 로또와 당첨 번호를 비교하여 **일치하는 번호의 개수**를 계산한다.
    - 당첨 번호와의 **일치 개수 및 보너스 번호 일치 여부**에 따라 1등부터 5등까지 등수를 판별한다.

7. **당첨 내역 출력**
    - 각 등수에 해당하는 당첨 로또 개수를 출력한다.

8. **수익률 계산 및 출력**
    - 총 당첨금액을 구입 금액으로 나눈 값을 이용해 **수익률을 계산**한다.
    - 수익률은 **소수점 둘째 자리에서 반올림**하여 출력한다.

9. **예외 처리**
    - 모든 입력은 **유효성 검사**를 수행하고, 잘못된 입력값이 주어지면 `[ERROR]`로 시작하는 메시지를 출력한 후 재입력을 받는다.
    - 예외는 명확한 유형으로 구분하여 처리한다.

# 🎰 로또 발매기 기능 요구 사항

### 1. 로또 번호 범위
- 로또 번호는 **1~45** 사이의 숫자로 구성된다.

### 2. 로또 발행
- 1장의 로또에는 **중복되지 않는 6개의 숫자**가 포함되어야 한다.

### 3. 당첨 번호 추첨
- 당첨 번호는 **중복되지 않는 6개의 숫자**와 **보너스 번호 1개**로 구성된다.

### 4. 당첨 기준 및 상금

로또의 당첨 기준과 상금은 아래 표와 같다:

| 순위 | 당첨 조건             | 상금             |
|----|-------------------|----------------|
| 1등 | 6개 번호 일치          | 2,000,000,000원 |
| 2등 | 5개 번호 + 보너스 번호 일치 | 30,000,000원    |
| 3등 | 5개 번호 일치          | 1,500,000원     |
| 4등 | 4개 번호 일치          | 50,000원        |
| 5등 | 3개 번호 일치          | 5,000원         |

### 5. 로또 구입 금액에 따른 로또 발행
- 로또 **1장 가격은 1,000원**이다.
- 로또 구입 금액을 입력받아, **해당 금액만큼의 로또를 발행**한다.

### 6. 당첨 번호 및 보너스 번호 입력
- 당첨 번호와 보너스 번호를 사용자로부터 입력받는다.

### 7. 당첨 결과 확인
- 사용자가 구매한 로또 번호와 당첨 번호를 비교하여 **당첨 내역 및 수익률을 출력**한다.
- 수익률을 계산한 후, **로또 게임을 종료**한다.

### 8. 예외 처리
- 사용자가 잘못된 값을 입력할 경우 **IllegalArgumentException을 발생**시키고, `[ERROR]`로 시작하는 오류 메시지를 출력한다.
- 오류 발생 시 해당 부분부터 다시 입력받는다.
- **IllegalArgumentException** 및 **IllegalStateException**과 같은 명확한 예외를 사용하여 오류를 처리한다.

---

## ⚙️ 예외 처리 규칙
- 오류 메시지는 `[ERROR]`로 시작한다.
- 입력 값의 유효성을 검사하고, 잘못된 값 입력 시 적절한 예외를 발생시킨다.

--- 

### 💡 기타 유의 사항
- **사용자 친화적인 메시지**를 출력하여 입력 오류 시 다시 입력을 유도한다.
- **클린 코드 원칙**을 준수하여 유지보수가 용이한 코드를 작성한다.

--- 

### 📈 수익률 계산 방식
- **수익률** = `(당첨금의 합 / 로또 구입 총액) * 100`

# 🎰 로또 발매기 입출력 요구 사항

## 🖥 입력

1. **로또 구입 금액 입력**
    - 로또 구입 금액을 입력 받는다. 구입 금액은 **1,000원 단위**로 입력 받으며, **1,000원으로 나누어 떨어지지 않는 경우 예외 처리**한다.
    - **예시 입력**: `14000`

2. **당첨 번호 입력**
    - 당첨 번호를 입력 받는다. **번호는 쉼표(,)로 구분**한다.
    - **예시 입력**: `1,2,3,4,5,6`

3. **보너스 번호 입력**
    - 보너스 번호를 입력 받는다.
    - **예시 입력**: `7`

---

## 🖨 출력

1. **발행한 로또 수량 및 번호 출력**
    - 발행한 로또의 수량과 각 로또 번호를 출력한다. **로또 번호는 오름차순으로 정렬**하여 보여준다.
    - **예시 출력**:
      ```
      8개를 구매했습니다.
      [8, 21, 23, 41, 42, 43]
      [3, 5, 11, 16, 32, 38]
      [7, 11, 16, 35, 36, 44]
      [1, 8, 11, 31, 41, 42]
      [13, 14, 16, 38, 42, 45]
      [7, 11, 30, 40, 42, 43]
      [2, 13, 22, 32, 38, 45]
      [1, 3, 5, 14, 22, 45]
      ```

2. **당첨 내역 출력**
    - 각 당첨 조건에 해당하는 로또 개수를 출력한다.
    - **예시 출력**:
      ```
      3개 일치 (5,000원) - 1개
      4개 일치 (50,000원) - 0개
      5개 일치 (1,500,000원) - 0개
      5개 일치, 보너스 볼 일치 (30,000,000원) - 0개
      6개 일치 (2,000,000,000원) - 0개
      ```

3. **수익률 출력**
    - 수익률은 **소수점 둘째 자리에서 반올림**하여 출력한다. (예: `100.0%`, `51.5%`, `1,000,000.0%`)
    - **예시 출력**: `총 수익률은 62.5%입니다.`

4. **예외 상황 에러 메시지 출력**
    - 예외 상황 시 `[ERROR]`로 시작하는 에러 문구를 출력한다.
    - **예시 출력**: `[ERROR] 로또 번호는 1부터 45 사이의 숫자여야 합니다.`

# 📌 프로그래밍 요구 사항

## 🔖 프로그래밍 요구 사항 1

1. **JDK 21 버전에서 실행 가능해야 한다.**
2. 프로그램 실행의 시작점은 `Application`의 `main()` 메서드이다.
3. `build.gradle` 파일은 변경할 수 없으며, 제공된 라이브러리 이외의 외부 라이브러리는 사용하지 않는다.
4. 프로그램 종료 시 `System.exit()`를 호출하지 않는다.
5. 프로그래밍 요구 사항에서 달리 명시하지 않는 한 **파일, 패키지 등의 이름을 바꾸거나 이동하지 않는다.**
6. 자바 코드 컨벤션을 지키며 프로그래밍한다.
7. **기본적으로 Java Style Guide를 원칙**으로 한다.

---

## 🔖 프로그래밍 요구 사항 2

1. **Indent(인덴트, 들여쓰기) Depth는 최대 2까지 허용**한다. 3을 넘지 않도록 구현한다.
   - 예: `while`문 안에 `if`문이 있는 경우 인덴트 깊이는 2이다.
   - **힌트**: 인덴트 깊이를 줄이는 좋은 방법은 함수를 분리하는 것이다.

2. **3항 연산자를 쓰지 않는다.**

3. 함수(또는 메서드)가 한 가지 일만 하도록 최대한 작게 만들어라.

4. **JUnit 5와 AssertJ를 이용해 테스트 코드를 작성하여 구현한 기능을 검증**한다.
   - 테스트 도구 사용법이 익숙하지 않다면 아래 문서를 참고한다:
      - [JUnit 5 User Guide](https://junit.org/junit5/docs/current/user-guide/)
      - [AssertJ User Guide](https://assertj.github.io/doc/)
      - [AssertJ Exception Assertions](https://assertj.github.io/doc/#assertj-core-exception-assertions)
      - [Guide to JUnit 5 Parameterized Tests](https://www.baeldung.com/parameterized-tests-junit-5)

---

## 🔖 프로그래밍 요구 사항 3

1. **함수(또는 메서드)의 길이는 최대 15라인**을 넘지 않도록 구현한다.
2. 함수(또는 메서드)는 한 가지 일만 하도록 구현한다.
3. **`else` 예약어를 쓰지 않는다.**
   - `else` 대신 `if` 조건절에서 값을 `return`하는 방식으로 구현한다.
   - `switch/case`도 허용하지 않는다.
4. **Java Enum을 사용하여 프로그램을 구현**한다.
5. 구현한 기능에 대한 **단위 테스트를 작성**한다. 단, UI(`System.out`, `System.in`, `Scanner`) 로직은 제외한다.
   - 단위 테스트 작성이 익숙하지 않다면 `LottoTest`를 참고하여 학습한 후 작성한다.

# Lotto 클래스 요구 사항

- **제공된 `Lotto` 클래스를 사용하여 구현**해야 한다.
- `Lotto`에 `numbers` 이외의 **필드(인스턴스 변수)를 추가할 수 없다.**
- `numbers`의 **접근 제어자인 `private`을 변경할 수 없다.**
- `Lotto`의 **패키지를 변경할 수 있다.**

        if (numbers.size() != 6) {
            throw new IllegalArgumentException("[ERROR] 로또 번호는 6개여야 합니다.");
        }
